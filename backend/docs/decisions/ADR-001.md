# ADR-001: Package manager: UV

## Estado
Aceptado

## Fecha
2026-03-30

## Contexto
Se requiere instalacion reproducible y rapida del entorno Python del backend.

## Decision
Adoptar `uv` como package manager y runner principal.

## Alternativas consideradas
| Alternativa | Por que se descarto |
|---|---|
| pip + venv | Menor ergonomia y velocidad en lock/sync |
| poetry | Mayor complejidad operativa para este equipo |

## Consecuencias
### Positivas
Instalacion rapida y consistente.
### Negativas / trade-offs
Curva de adopcion inicial.
### Neutrales
No cambia arquitectura de dominio.

## Criterios de aceptacion de la decision
`uv sync` instala dependencias sin pasos adicionales.

## Referencias
Stack base del proyecto.
