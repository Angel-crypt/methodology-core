# ADR-014: Magic Link hash + TTL + uso unico

## Estado
Aceptado

## Fecha
2026-03-30

## Contexto
El onboarding seguro requiere token temporal no reutilizable.

## Decision
Implementar Magic Link de un solo uso, con TTL y almacenamiento de hash del token.

## Alternativas consideradas
| Alternativa | Por que se descarto |
|---|---|
| Guardar token en claro | Riesgo de exposicion directa |
| Token reutilizable | Aumenta superficie de abuso |

## Consecuencias
### Positivas
Mejor postura de seguridad en onboarding.
### Negativas / trade-offs
Mayor complejidad de soporte cuando expira token.
### Neutrales
Requiere endpoint de regeneracion administrativo.

## Criterios de aceptacion de la decision
Segundo uso o expiracion retorna `410`; DB no almacena token en claro.

## Referencias
Arquitectura aprobada de identidad.
