# ADR-005: Testing: pytest + pytest-asyncio + httpx

## Estado
Aceptado

## Fecha
2026-03-30

## Contexto
Se requiere piramide de tests para codigo async y endpoints HTTP.

## Decision
Adoptar pytest, pytest-asyncio y httpx AsyncClient.

## Alternativas consideradas
| Alternativa | Por que se descarto |
|---|---|
| unittest nativo | Menor ergonomia y ecosistema |

## Consecuencias
### Positivas
Cubre unit, integration y contract en un marco comun.
### Negativas / trade-offs
Configuracion de fixtures async mas elaborada.
### Neutrales
No altera API publica.

## Criterios de aceptacion de la decision
Suite async corre estable y reproduce criterios de aceptacion.

## Referencias
Estrategia de testing del proyecto.
