# ADR-016: Base de datos: PostgreSQL

## Estado
Aceptado

## Fecha
2026-03-30

## Contexto
Se necesita motor relacional robusto y compatible con stack async definido.

## Decision
Adoptar PostgreSQL para todos los entornos del backend.

## Alternativas consideradas
| Alternativa | Por que se descarto |
|---|---|
| SQLite dev | Menor paridad con entorno objetivo |
| MySQL | No es el estandar acordado para el proyecto |

## Consecuencias
### Positivas
Consistencia entre desarrollo y despliegue.
### Negativas / trade-offs
Dependencia de servicio DB en local y CI.
### Neutrales
No cambia contratos de API.

## Criterios de aceptacion de la decision
Migraciones Alembic ejecutan en PostgreSQL sin divergencias.

## Referencias
Respuesta inicial de alcance y restricciones.
