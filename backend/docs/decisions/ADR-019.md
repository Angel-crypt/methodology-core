# ADR-019: Metodologia SDD + TDD

## Estado
Aceptado

## Fecha
2026-03-30

## Contexto
Se requiere control de deriva entre contratos, schemas y comportamiento implementado.

## Decision
Aplicar flujo inamovible por endpoint: contrato XML -> schema Pydantic -> tests -> implementacion.

## Alternativas consideradas
| Alternativa | Por que se descarto |
|---|---|
| Implementar primero y testear despues | Mayor riesgo de desviacion de contrato |

## Consecuencias
### Positivas
Mayor trazabilidad y calidad de entrega.
### Negativas / trade-offs
Mayor disciplina y costo inicial por endpoint.
### Neutrales
Compatible con CI por contract tests.

## Criterios de aceptacion de la decision
Todo endpoint nuevo tiene criterio+test antes de cerrar implementacion.

## Referencias
Lineamiento metodologico del proyecto.
