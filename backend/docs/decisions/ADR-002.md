# ADR-002: ORM: SQLAlchemy 2.x async

## Estado
Aceptado

## Fecha
2026-03-30

## Contexto
El backend define ejecucion async por defecto y requiere ORM maduro.

## Decision
Usar SQLAlchemy 2.x async para acceso a datos.

## Alternativas consideradas
| Alternativa | Por que se descarto |
|---|---|
| SQLModel | Menor control fino en este alcance |
| ORM sync | Incompatible con lineamiento async-first |

## Consecuencias
### Positivas
Modelo robusto y ecosistema amplio.
### Negativas / trade-offs
Tipado de relaciones requiere ajustes en mypy.
### Neutrales
No altera contratos HTTP.

## Criterios de aceptacion de la decision
Repositorios async operan con sesiones async y pruebas de integracion estables.

## Referencias
Stack base definitivo.
