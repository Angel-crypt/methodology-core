# ADR-004: Type checking: mypy strict adaptado

## Estado
Aceptado

## Fecha
2026-03-30

## Contexto
El proyecto exige type hints obligatorios y control de calidad estatico.

## Decision
Usar mypy en modo strict adaptado a FastAPI + SQLAlchemy.

## Alternativas consideradas
| Alternativa | Por que se descarto |
|---|---|
| Sin type checker | Riesgo alto de errores en runtime |
| pyright unicamente | Se prioriza mypy por stack acordado |

## Consecuencias
### Positivas
Reduce errores de tipos en capas criticas.
### Negativas / trade-offs
Requiere suppressions justificadas en relaciones ORM.
### Neutrales
No afecta performance de runtime.

## Criterios de aceptacion de la decision
CI falla ante errores de tipos no justificados.

## Referencias
Stack base.
